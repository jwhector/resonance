The two tensions — resolved inside ADR-0017

These are acceptance conditions on step 1, not afterthoughts.

1 · "boundary" vs "seam"

The skill says avoid "boundary"; ADR-0003 is literally package-boundaries-as-context-boundaries.

Keep "boundary" for ADR-0003's package / context / ownership sense; introduce "seam" for the finer-grained location of an interface. A package boundary is one kind of seam. Do not rename ADR-0003 — add the mapping in the new ADR.

2 · model-ahead ports vs "two adapters = real seam"

Resonance models some seams before their real adapter exists (StoragePort deferred, PaymentsPort stubbed).

Justified: the test fake is the second adapter and the variance is known/imminent, so "known future variance + a test fake" satisfies the two-adapter rule. A truly single-adapter, no-known-variance seam is the one to avoid.

Acceptance criteria

  • ✅ ADR-0017 Accepted, has a README row, and is indexed by a mulch architecture reference record.
  • ✅ ADR records both resolutions (boundary/seam; model-ahead ports) explicitly.
  • ✅ SKILL.md's "provisional" note updated to reference the ratified ADR.
  • ✅ conventions.md has a "Module design" section (glossary, 3 interface questions, deletion test, interface-as-test-surface, one/two-adapter rule, 3 testability rules).
  • ✅ Root CLAUDE.md has the one-line deep-module pointer.
  • ✅ Each of the 4 recipes frames its unit as a module with a small interface, in the exact vocabulary.
  • ✅ Planning guidance + prompt templates instruct agents to decompose into deep modules and name each seam.
  • ✅ A "module depth" review checklist exists and is referenced from review guidance.
  • ✅ pnpm check:workspace passes; one consistent vocabulary, no term drift.

Risks & guards

  • ⚠️ Vocabulary drift (boundary vs seam) — state the ADR-0003 mapping crisply; don't rename ADR-0003.
  • ⚠️ One-fact-one-home (ADR-0016) — ADR holds the why, conventions holds the rule, others link. No duplicated rule text.
  • ⚠️ Over-scoping — formalization + vocabulary only; no package/port/code changes.
  • ⚠️ Review dimension has no repo home today — no-mistakes & /code-review are external plugin skills; it must land as a repo-owned checklist that review guidance references.
  • ⚠️ Dangling "provisional" — SKILL.md's note must be updated when the ADR lands (folded into step 1).
